# 공통 규칙

## 사실 점검 (Fact Check)
- 모든 데이터, 수치, URL, 브랜드명은 **최소 2회 이상** 교차 검증할 것
- 리서치 결과물에 출처(URL, 날짜)를 반드시 명시
- 불확실한 정보는 "추정", "미확인" 표기 후 사용자에게 알릴 것
- 기존 코드/파일을 수정하기 전에 반드시 현재 상태를 먼저 확인

## 오류 점검 (Error Check)
- 코드 작성 후 반드시 빌드/타입 체크 실행 (`npx tsc --noEmit` 또는 `npx next build`)
- git commit 전에 `git status`로 변경사항 확인
- 배포 전 로컬에서 동작 확인
- HTML 제안서는 브라우저에서 열어 깨지는 부분 없는지 확인
- 외부 이미지 URL 사용 시 onerror 폴백 처리 필수
