## 0. 한 줄 정의

**"5분 게임처럼 푸는 진단(BEST의 입구) → 컨설팅 시뮬레이션 결과지(얼터의 출구)"** 하이브리드.
인트로/폼은 도파민으로 데려오고, 결과지는 "내가 못 채우는 빈 칸"을 보여주며 1:1 상담을 트리거한다.

---

## 1. v1(현행) 이탈 원인 가설

| 단계 | 가설 | 근거 |
|------|------|------|
| 인트로 → 폼 진입 | 가치제안이 "마케팅 성숙도"라는 추상 개념 | "성숙도"는 대표 입장에서 절박함 X |
| 폼 진행 중 | 6문항만 있는데도 "정답이 있는 시험" 인상 | 4점 척도 = 평가받는 느낌, 게임 X |
| 결과 페이지 → 상담 신청 | 점수+등급+영역별 조언이 너무 친절·완결적 | 빈 칸이 없어서 "이 정도면 됐다" |

**핵심 진단:** 결과지가 너무 친절해서 다음 단계가 안 필요해 보인다. **1:1 전환 트리거는 "결과지의 의도된 결핍"이다.**

---

## 2. 핵심 전략 — 하이브리드 3층

```
[Layer 1: 입구]   BEST 게임화 톤
  → "당신 회사 마케팅 DNA는?" 5분 / 카드형 1문항 / 자동진행 / 진행바
  → 도파민·낮은 진입장벽으로 이탈 차단

[Layer 2: 분류]   유형 + 단계 2축 분류 (BEST식 코드 부여)
  → 결과 화면에 "OO형" 코드 + 별명 부여 (공유 가능성 ↑)
  → 단, 코드는 "정체성"이 아니라 "현재 회사 상태 진단표"로 프레이밍

[Layer 3: 출구]   얼터 컨설팅 리포트 톤
  → PART 1: 우리 회사 진단표 (초상)
  → PART 2: "우리가 봤을 때" 위험 신호 + 갭 (위기감)
  → PART 3: 1:1 상담에서 풀어드릴 핵심 1가지 (트리거)
```

## 3. 진단명 / 포지셔닝

**진단명 (가칭, 변경 가능):** **"브랜드 성장 처방전 — 우리 회사 마케팅, 진짜 어디가 문제일까"**

- "성숙도"(추상) → "처방전"(구체적·실행 약속)으로 변경
- 부제는 대표가 머릿속에 항상 갖고 다니는 질문 그대로 인용

**한 줄 가치제안:**
> "5분이면 우리 회사 마케팅의 진짜 병목과, 다음 한 수가 나옵니다."

**사회적 증거 카피 (인트로):**
- ✓ 스타트업 대표 N명이 받은 처방전
- ✓ 평균 3분 30초 소요
- ✓ 결과는 즉시, 심화 처방은 24시간 이내

---

## 4. 진단 폼 설계

### 4-1. 진행 UX (BEST 차용)
- **카드형 1문항씩** + 자동 다음(300ms 딜레이)
- 상단 진행바: `01 / 12` 표기
- 키보드 단축키 1/2/3/4
- "← 이전" 버튼
- 다크 테마 + 오렌지 액센트 (브랜드라이즈 기존 시스템 유지)

### 4-2. 질문 구성 (12문항, 5분 → 3분 30초)

> **⚠️ 5/6 override 알림**
> 아래 4축(F/T × I/D × C/P/S/N × Z/G/X) 분류는 **5/4 시점 v1 가정**.
> 5/6 결정으로 [12-1] 2축(F/T × S/G) 4유형 / [12-2] 영역 6항목 v2로 압축됨.
> 실제 v2 빌드 12문항은 `diagnosis-v2/specs/2026-05-06-questions-v1.md` 참조.

기존 6문항(영역 진단)은 유지하되, **분류용 4문항을 앞에 두어 "내 회사 정체성"을 만든다.**

```
[A. 정체성 분류 4문항] — 결과 코드 산출용
A1. 우리 회사 마케팅의 의사결정자는?       → 축 1: 대표주도(F) vs 팀주도(T)
A2. 마케팅 예산 운영 방식은?              → 축 2: 직감집행(I) vs 데이터배분(D)
A3. 우리 회사가 가장 자주 쓰는 무기는?     → 축 3: 콘텐츠(C) vs 광고(P) vs 영업(S) vs 채널(N)
A4. 지금 회사의 성장 단계는?              → 축 4: 0→1(Z) vs 1→10(G) vs 10→100(X)

[B. 영역 진단 6문항] — 기존 v1 유지 (점수산출용)
B1. 브랜드 일관성 (1-4)
B2. 성과 추적 (1-4)
B3. 콘텐츠 체계 (1-4)
B4. 고객 이해 (1-4)
B5. 성장 병목 (1-4)
B6. 실행 속도 (1-4)

[C. 처방 트리거 2문항] — 결과지 PART 3 맞춤화
C1. 가장 시급한 과제 (복수선택, 4지선다)  → v1 Q7 유지
C2. 한 줄로, 지금 가장 답답한 것은?       → 자유서술 30~80자 (★ 신규)
```

**C2가 핵심:** 자유서술 한 줄로 받은 "진짜 고민"이 PART 3 트리거 카피의 변수로 들어간다. 얼터 PDF의 "본인 직접 경험" 정보 흡수와 같은 역할.

### 4-3. 결과 코드 (BEST식 4글자 + 단계)

`{F/T} {I/D} {C/P/S/N} {Z/G/X}` = 예: `FDCG` = "대표주도·데이터·콘텐츠·1→10단계"
→ 별명 부여: "FDCG · **데이터로 콘텐츠를 짓는 직감장인**"

별명 16~24개 사전 작성. 마케터 DNA처럼 정체성 부여 + 공유 가능.

## 5. 결과지 설계 (★ 핵심 차별점)

### 5-1. 전체 구조 (얼터 PART 구조 차용)

```
[Hero]
  코드(FDCG) + 별명 + 한줄 캐치프레이즈
  → 즉시 캡처/공유 욕구 자극

[PART 1] 우리 회사 진단표 — "지금 이렇게 보입니다"
  ├─ 6영역 점수 카드 (낮은 순 정렬, v1 자산 재활용)
  ├─ 종합 점수 (24점 만점) — 단, 등급 라벨 노출 X
  └─ 한 줄 정성 진단: "FDCG는 대체로 이런 함정에 잘 빠져요"

[PART 2] 위험 신호 진단 — "이대로 가면 이렇게 됩니다" ★ 신규
  ├─ 탈락 위험도 (낮음 / 중간 / 높음)
  ├─ 5개 항목 처방 우선순위
  │   - 각 항목: [강점/보완 필요/위험] 라벨 + 한 줄 진단
  ├─ "이런 점이 우리 진단가의 눈에 보입니다" 카드 3장
  │   - C2(자유서술) 키워드를 인용해 개인화
  └─ 격차 시각화: 현재 N점 → 처방 적용 시 예상 N점 (얼터의 22%→81% 차용)

[PART 3] 다음 한 수 — "1:1에서 풀어드릴 핵심 1가지" ★ 트리거
  ├─ 우선순위 1번 처방의 헤드라인 + 첫 단락만 공개
  ├─ "이 처방을 우리 회사에 맞게 정밀화하려면, 30분 1:1이 필요합니다"
  ├─ 1:1에서 받게 될 것 3가지 (구체)
  │   - 우리 회사 데이터 기반 우선순위 도출
  │   - 4주 실행 로드맵 1장
  │   - 다음 캠페인의 KPI 기준선
  └─ [30분 무료 1:1 처방 받기] CTA — 오렌지 풀폭

[하단]
  공유 카드 (OG 이미지 자동생성: 코드 + 별명 + 점수)
  "내 처방전 결과 공유하기" → 카톡/X/링크복사
```

### 5-2. 의도된 결핍 3가지 (이탈 → 트리거 전환)

기존 v1이 결과지를 "충분히 친절하게" 만들어 이탈을 부른 지점을 의도적으로 비운다.

| v1 (이탈 발생) | v2 (트리거) |
|----------------|-------------|
| 영역별 다음레벨 조언 풀세트 제공 | **PART 3에서 1번 처방의 첫 단락만** 공개 |
| 등급(D/C/B/A) 즉시 라벨링 | **등급 라벨 제거**, 대신 처방 갯수로 표현 ("우리가 본 위험 신호 3건") |
| "직접 하셔도 됩니다" 면책성 카피 | **"이건 직접 하기엔 손이 안 맞습니다"** 정직한 한계 명시 |

★ 단, "세일즈 냄새 안 나야 한다"는 모수 블루프린트 원칙은 유지. PART 3은 **"이 처방을 정밀화하려면"** 톤이지 **"우리한테 맡기세요"** 톤이 아니다.

## 6. 트리거 → 1:1 전환 메커니즘

| 장치 | 위치 | 작동 원리 |
|------|------|----------|
| 코드 + 별명 | Hero | 정체성 부여 → 공유로 유입 재생산 |
| 위험도 시각화 | PART 2 | 위기감 → 행동 동기 |
| 격차 비교 (현재 N점 → 보완 시 N점) | PART 2 | 갭 가시화 → "지금 그대로면 손해" |
| C2 자유서술 인용 | PART 2 | 개인화 → "내 얘기 같다" |
| 처방 1번 단락 공개 후 컷오프 | PART 3 | 호기심 갭 → "나머지 알고 싶다" |
| 1:1에서 받게 될 것 3가지 | PART 3 | 구체적 약속 → 시간 투자 정당화 |

## 12. 설계 변경 (5/6 워크샵 반영)

5/6 우성민 컨펌·시뮬레이션 후 본문 위 섹션에 우선하는 결정사항.

### 12-1. 유형 압축: 4유형 (2축 × 2)

| 축 | 양극단 | 코드 |
|----|-------|------|
| 축1 — **마케팅 의사결정자** | 대표가 직접 / 담당자가 위임 | F / T |
| 축2 — **마케팅 인력 상태** | 대표 혼자 / 팀 있음 | S / G |

→ 4유형: **FS** / **FG** / **TS** / **TG**
→ 매출·업종은 분류 변수에서 제외, PART 2 처방 개인화 변수로만 사용
→ "0→1, 1→10" 등 스타트업 내부 어휘는 폐기 (대표가 즉답할 수 있는 표현으로 통일)

### 12-2. 영역 진단 6항목을 2026년 버전으로 재구성

| 기존 (v1) | 신 v2 |
|-----------|------|
| 브랜드 일관성 | 브랜드 일관성 |
| 성과 추적 | **데이터·대시보드 성숙도** ★ 진화 |
| 콘텐츠 체계 | 콘텐츠 체계 |
| 고객 이해 | 고객 이해 |
| 성장 병목 | **AI·자동화 도입도** ★ 신규 |
| 실행 속도 | 실행 속도 |

→ AI·자동화 1~2점 시 PART 2 "트렌드 격차 카드" 강제 노출 ("2026년 시장에서 12개월 내 격차로 작용")

### 12-3. 결과지 1차 = 4섹션 압축 (BEST 7섹션은 3차/8월)

