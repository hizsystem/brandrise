# 견적서 포맷 규칙 (브랜드라이즈 표준)

> 제정: 2026-06-03 · 개정: 2026-06-11 (인디고 클린 통합표준 — 클라이언트별 컬러 폐기)
> 트리거: "견적서 만들어", "{회사명} 견적", "미팅 후 견적 정리"
> 정본 경로: `brandrise-pages/quote/{slug}/` (HTML) · 구글시트(고액)
> 템플릿: `quote/_template/quote-template.html` (HTML) · `quote/_template/quote-builder-template.gs` (시트)
> 전체 문서 체계: 루트 `TEMPLATES.md` 참조

미팅이 끝나고 **회사명 · 스코프 · 금액대** 3가지가 정해지면, 아래 규칙에 따라 자동으로 견적서를 만든다.

---

## 1. 규모별 분기 (1줄 룰)

| 견적 규모 (VAT 별도) | 포맷 | 안(案) 수 | 레퍼런스 |
|---|---|---|---|
| **1,000만원 이하** | HTML (`quote/{slug}/index.html`) | 1~2안 | 오어브랜드 (300/700) |
| **1,000만원 초과** | 구글시트 (강훈 5블록) | 2안 (코어/풀) | 심박 (3,000/4,800), 강훈목장 (5,400/9,800) |

- HTML은 가벼운 단건·디자인성 견적. **`quote/_template/quote-template.html` 복제 — 인디고 클린 통합표준** (⛔ 클라이언트별 컬러 금지, 2026-06-11 개정). 표는 항목/계약견적만.
- 구글시트는 파트너십성 고액 견적. **`quote/_template/quote-builder-template.gs`** (강훈 워크북 레이아웃 미러, 인디고 밴드). 1,000만 초과 시 HTML은 브리핑·발송용 요약 겸용 가능.

---

## 2. 강훈 5블록 골격 (구글시트 표준)

표 컬럼: **구분 / 항목·내용 / 단가 / 수량 / 계약견적** — ⛔ M/M 노임단가표(투여율·1일인건비·제경비) 금지.

블록은 프로젝트에 맞게 0~4에서 취사:

| 블록 | 내용 |
|---|---|
| 0) 브랜드 기획 | 포지셔닝·메시지·브랜드북 (미니 ↔ 풀) |
| 1) BI·패키지 디자인 | 로고·패키지·셀시트 |
| 2) 자사몰 세팅 | 카페24 IA·UI·상세 (해당 시) |
| 3) SNS·인플루언서 | 무드보드·콘텐츠 기획·시딩 |
| 4) 추가(촬영) | 제품/목장 촬영 — 보통 옵션 |

각 블록 끝에 **소계**, 맨 아래 **합계(VAT 별도)**. 좌상단 **파트너십 방향 3줄**, 하단 **비고**(VAT·촬영별도·인쇄별도·실비별도·결제조건·자료회신 시 조정).

**2안 설계 원칙**: 코어 = "지금 가장 시급한 것부터(하한 앵커)", 풀 = "토대를 제대로". 금액 차이는 블록 깊이(기간·산출물)로 만든다.

---

## 2-1. 마케팅 방향 가안 (견적표 우측 G열) — 브랜드 마케팅 이해도 낮은 대표용

오늘의평야·쿠엔즈버킷 표준. 견적 용어를 모르는 대표가 "이게 무슨 작업인지" 1초에 이해하도록, 견적표 **오른쪽에 블록별 가안**을 붙인다. (2026-06-08 심박 첫 적용)

**블록별 3단 구조** (또는 한 셀 압축형):
- **개선지점)** 현재 무엇이 문제인가 — ⚠️ 대표 자존심 고려: 라벨은 "문제"보다 **"현재"**
- **목표)** 무엇을 만드는가 (2~3줄)
- **운영방식) / 타겟 브랜드)** 어떻게 / 닮을 브랜드

**핵심**: 현재 모습 **→** 되고 싶은 모습을 화살표 한 줄로 ("쌀 판매 사이트 → 프리미엄 라이프스타일 편집샵"). **미팅 발언을 그대로 옮기는 게 가장 강하다** (심박: 김지엽 "샤인머스켓 포도에서 분리").

**예시 브랜드** = ②우리 작업사례(신뢰 증명) + ③벤치마크 롤모델(방향). ⛔ **①직접 경쟁사는 견적서에 넣지 않는다**("쟤가 더 낫다" 역효과) — 내부 분석용만.

**블록 성격에 예시 매칭**: 패키지=패키지·입점 사례(타이거모닝) / 자사몰=실제 사이트 좋은 브랜드(요헤미티) / 인스타=콘텐츠 강한 브랜드(베지어트).

**압축형 ↔ 3단형 선택**: 핵심 블록(대표 페인 직결, 보통 인스타·패키지)은 3단으로 길게, 나머지는 압축형 한 셀.

## 3. 심플화 = 구매자가 사는 것을 바꾸기

견적 단순화의 핵심은 표를 줄이는 게 아니라 **산정 로직을 숨기고 "무엇을 주는가 + 얼마"만 보여주는 것**.
- 외주 감각 클라이언트(크몽 경험자)에는 **"1회 디자인 구매 vs 계속 쓰는 자산 구매"** 대비 1박스로 가치 점프를 정당화.
- 2안 제시 = 앵커링. 풀안이 코어안의 합리성을 만들어 준다.

## 5. 빌더 엔진 재사용 (구글시트 자동 생성) — 다음 브랜드 적용법

**정본 엔진 = `quote/_template/quote-builder-template.gs`의 `renderQuote_()`** (브랜드 무관 범용 렌더, 심박 빌더에서 추출·2026-06-11 템플릿화). 강훈 레이아웃·블록 세로병합·KPI 2단·추가 논의 영역·우측 마케팅 가안(G열)을 전부 포함한다.

**새 브랜드 적용 = 엔진은 그대로 복사, 데이터 객체만 교체:**
1. `KPI` / `NOTES` — 파트너십 3~4줄 + 비고
2. `DISCUSS_CORE` / `DISCUSS_FULL` — 추가 논의 영역(가격 없는 후속 항목)
3. `P_xxx` — 마케팅 가안 텍스트 (blocks와 **동일 순서**)
4. `core` / `full` 객체 — `blocks[]` · `total` · `discuss` · `proposals[]`
5. `renderQuote_` 호출 2줄 — 탭명·금액 라벨

→ 강훈 복제본 시트 → 확장 → Apps Script → 붙여넣기 → `build{Brand}Quote` 실행.

⚠️ 엔진 수정(레이아웃·서식 로직)은 모든 브랜드에 영향 → 한 곳(최신 빌더)에서만 고치고 전파.
