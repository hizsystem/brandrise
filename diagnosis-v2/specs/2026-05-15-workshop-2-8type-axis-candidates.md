# 워크샵 2 — 8유형 확장 새 축 선택

## 의사결정 한 줄

현재 2축(F/T × S/G) → 4유형. 새 1축을 추가해 3축 = 8유형으로 확장한다. 어느 축을 추가할지 결정한다.

## 새 축 후보 3안 (Claude 추천 단일 경로 → **B. PMF**)

### A안 — 시간성 축 (Acute ↔ Chronic)

**의미**: 답답함이 발생한 시점이 최근 6개월 이내(Acute) vs 1년 이상 누적(Chronic).

| 코드 | 의미 | 처방 차별 |
|------|------|----------|
| FSA / FSC | 1인 다역, 급성/만성 | Acute = 빠른 진단·격자식 KPI / Chronic = 구조적 리셋 |
| TSA / TSC | 위임형, 급성/만성 | Acute = 마케터 KPI 재정의 / Chronic = 검증 시스템 재설계 |
| FGA / FGC | 광고 정체, 급성/만성 | 동일 패턴 |
| TGA / TGC | 다음 단계, 급성/만성 | 동일 패턴 |

**장점**: 답답함의 깊이를 잡음. 처방 우선순위에 직결.
**단점**: "시간"이 추상적. 사용자가 정확히 매핑 어려울 수 있음.

### B안 — PMF 축 (Pre-PMF ↔ Post-PMF) ★ Claude 추천

**의미**: 제품-시장 적합도(Product-Market Fit) 발견 전 vs 후.
- Pre-PMF: 매출이 계속 흔들림, 반복 구매 모름, 핵심 고객이 누군지 가설 단계
- Post-PMF: 매출이 안정적, 반복 구매 발생, 핵심 고객 명확

| 코드 | 의미 | 처방 차별 |
|------|------|----------|
| FSP / FSM | 1인 다역, Pre/Post-PMF | Pre = 가설 검증 마케팅 / Post = 확장 마케팅 |
| **TSP ★ / TSM ★** | **위임형, Pre/Post-PMF** | **Pre = 마케터에게 가설 검증 위임 (왜 안 보이는지 못 잡음) / Post = 확장 KPI 재정의 (확장이 안 됨)** |
| FGP / FGM | 광고 정체, Pre/Post-PMF | Pre = 광고로 PMF 가짜 신호 / Post = 광고 효율 한계 |
| TGP / TGM | 다음 단계, Pre/Post-PMF | Pre = 확장 가설 부재 / Post = 다음 시장 가설 부재 |

**장점**:
- 모수 페르소나(시드~프리A 대표)가 가장 자주 헷갈리는 단계 변수
- 처방의 본질이 PMF 전후로 완전히 다름 — sophistication 강함
- TS 골든이 Pre-PMF TS / Post-PMF TS로 분기되어 정밀도 ↑

**단점**: "PMF"라는 용어가 진단자에게 친숙해야 함 → 폼 질문에서 자연어로 풀어 물어보면 해결 가능.

### C안 — 채널 축 (Owned ↔ Paid)

**의미**: 마케팅 의존 채널이 자체 채널(블로그·SNS·SEO) 중심인지, 광고(Meta·구글) 중심인지.

| 코드 | 의미 | 처방 차별 |
|------|------|----------|
| FSO / FSP | 1인 다역, Owned/Paid 의존 | Owned = 콘텐츠 시스템 / Paid = 광고 최적화 |
| ... (동일 패턴 8유형) | | |

**장점**: 마케팅 처방의 본질에 가까움. 사용자 자기 인식 쉬움.
**단점**: F/T축과 일부 중복 (T형은 Paid 의존이 흔함). 차별성 약함.

## 추천 단일 경로: **B안 PMF**

이유 3가지:
1. **모수 페르소나 직격**: 시드~프리A 단계 회사가 가장 자주 막히는 변수가 PMF 전후. 진단 시점에 명확한 답이 나옴.
2. **처방 본질 차별화 큼**: Pre-PMF / Post-PMF는 같은 영역(데이터·콘텐츠·고객 이해)이라도 처방이 정반대. 24개 매트릭스를 48개로 확장 시 카피 작업 본질적 가치 큼.
3. **TS 골든 정밀화**: TS형이 Pre-PMF인지 Post-PMF인지에 따라 1:1 상담 전환률 더 분기 가능. Meta 광고 카피도 두 갈래로 정밀.

## 챗GPT 별창 워크샵 2 사용법

1. ChatGPT 별창 (GPT-5 또는 4o, 새 대화) 열기
2. **STEP 1**(System Prompt) 복붙 → "준비 완료" 응답 대기
3. **STEP 2**(8유형 별명 + 캐치 워크샵) 복붙 → 후보 토할 때까지 대기
4. 우성민 톤 평가로 후보 좁히기 ("FSP 별명은 후보3 확정, 캐치는 후보2 확정..." 식)
5. 별명 8 + 캐치 8 = 16줄 확정 → 본 Claude Code 세션 회신
6. **예상 시간: 45분** (4유형 → 8유형 확장이라 추가 30분)

## STEP 2 — 8유형 별명 + 캐치 워크샵 프롬프트

